Session 7

Functions in C++ and Reusing Code

🎯 Session Overview

Welcome to Session 7! In our last class, we learned how to repeat actions quickly using loops. But as you build bigger programs (like a full video game or a calculator app), your code will start getting very long and very messy.

Today, we will learn how to organize our code like real software engineers! You will learn:

  • What a function is and why programmers love them.
  • How to build your very own custom commands in C++.
  • How to "call" a function from inside your main program.
  • How to keep your code clean, organized, and easy to read.

⏪ Quick Recap: Session 6

In Session 6, we unlocked the power of loops:

  • We used the for loop to run code multiple times.
  • Loops are great for counting, printing patterns, and saving us from typing the same thing repeatedly.

However, loops only repeat code that is in one specific place. What if we want to run the same code at the beginning of the program, and then again at the very end? That's where Functions come in!

😩 The Problem: Messy, Repeating Code

Imagine you are building the TechTicket App. Every time a student clicks a button, you want to show a welcome message.

cout << "Welcome to TechTicket!" << endl;
// ... 100 lines of other code ...
cout << "Welcome to TechTicket!" << endl;
// ... 200 lines of other code ...
cout << "Welcome to TechTicket!" << endl;

This is a bad habit! If TechTicket changes its name to "CodeTicket", you have to hunt through hundreds of lines of code to find and change every single cout. There has to be a better way!

🤖 What is a Function?

A Function is like a mini-machine or a custom tool that you build inside your program. You teach this machine how to do one specific task.

Once you build the machine, you give it a name. Then, whenever you need that task done, you don't rewrite the code—you just call the machine by its name, and it does the work for you!

Fun fact: You have already been using a function this whole time! int main() is the master function where all C++ programs start.

🏗️ How to Build a Function

Let's build a mini-machine that prints our welcome message. We build it outside (above) the main() function.

void greet()
{
    cout << "Welcome to TechTicket!" << endl;
}

Let's break down this structure:

  • void → This is the function type. "Void" means empty. It tells C++ that this machine just does an action but doesn't hand any data back to us when it's done.
  • greet → This is the function name. You can name it almost anything! (Like sayHello, printWelcome, etc.)
  • () → The parentheses. This is where we will eventually put inputs (we'll learn that later!).
  • { ... } → The function body. This is the inside of the machine. The code here is what runs when the machine is turned on.

📞 Calling a Function

Just building the machine doesn't make it run. It will just sit there, turned off. To turn it on, we have to Call it from inside our main() program.

welcome.cpp 
#include <iostream>
using namespace std;

// 1. We build our function here
void greet()
{
    cout << "Welcome to TechTicket!" << endl;
}

int main()
{
    // 2. We "call" our function inside main()
    greet();  // Prints the message!
    greet();  // Prints it again!
    greet();  // Prints it a third time!

    return 0;
}

Notice how clean the main() function is now! If we want to change the message to "Welcome to C++!", we only have to change it in ONE place (inside the greet() machine), and it automatically updates everywhere!

👀 Visualizing the Function Call

When the computer is running main() and sees greet();, it presses "pause", jumps up to the greet function, runs all the code inside it, and then jumps right back to where it left off!

🍫 Real-Life Analogy: The Vending Machine

Think about a vending machine. Inside the machine, there is a complicated mechanical process: a coil spins, the snack falls, the trapdoor unlocks.

As a user, you don't need to know how the machine does all that. You just press the button labeled [A4].

  • The mechanical process inside is the Function Body { ... }.
  • The button [A4] is the Function Name.
  • Pressing the button is Calling the Function.

Every time you press the button, the machine repeats the exact same sequence for you!

Exercise 1: The Announcer

Create a function called announce() that prints: "Hello TechTicket Students!". Then, go into your main() function and call it 3 times.

Put void announce() { ... } ABOVE your int main(). Inside main(), type announce(); three times!

Exercise 2: The Motivator

Exercise 3: The Counter Machine

Functions can have loops inside them! Create a function called countToFive(). Inside its body, write a for loop that prints numbers from 1 to 5. Call it from main().

Put the for (int i = 1; i <= 5; i++) loop entirely between the curly braces of your new function!

🔥 Challenge Exercise: The Multiplier Tool

Create a function named printTable3().

Inside this function, use a loop to print out the multiplication table for the number 3 (e.g., 3x1=3, 3x2=6, up to 3x10=30).

Then, call this function from your main() program. If you do it right, your main() program will only be one line long, but it will print 10 lines of math to the screen!

void printTable3() {
    for(int i = 1; i <= 10; i++) {
        cout << "3 x " << i << " = " << (3*i) << endl;
    }
}

📝 Session Summary

  • A Function is a reusable block of code that performs a specific task.
  • Functions keep our code clean, organized, and prevent us from repeating ourselves.
  • Functions are built outside (usually above) the main() program.
  • The keyword void means the function just "does something" but doesn't hand data back.
  • To run a function, you must call it by typing its name followed by parentheses and a semicolon inside main() (e.g., greet();).
