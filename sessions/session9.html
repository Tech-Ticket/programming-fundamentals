🧠 How Programmers Solve Problems

When a programmer is asked to build a new feature (like a "Login System" or a "High Score Tracker"), they never start typing code immediately. They follow a 5-step master plan:

Let's use this 5-step method to solve three classic programming problems!

🔢 Example 1: Even or Odd Number

Step 1: The Problem: Ask the user to enter a number, and tell them whether that number is even or odd.

Steps 2 & 3: The Algorithm (Plain English):

  1. Create an integer box to hold the number.
  2. Ask the user to type a number and save it in the box.
  3. If the number divides perfectly by 2 (remainder is 0), then it's Even.
  4. Otherwise (else), it's Odd.

Step 4: The Code:

even_odd.cpp 
#include <iostream>
using namespace std;

int main()
{
    int number;
    cout << "Enter a number: ";
    cin >> number;

    // Using Modulus (%) to find the remainder!
    if(number % 2 == 0)
    {
        cout << "The number is even.";
    }
    else
    {
        cout << "The number is odd.";
    }

    return 0;
}

Step 5: Test! If we type 4, the math is 4 % 2. The remainder is 0, so it prints "Even". Perfect!

🏆 Example 2: Find the Largest Number

Step 1: The Problem: Ask the user for three different numbers. Figure out which one is the biggest and print it.

Steps 2 & 3: The Algorithm:

  1. Get three numbers from the user.
  2. Create a variable called largest and assume the 1st number is the biggest for now.
  3. Check the 2nd number. If it's bigger than largest, make it the new largest.
  4. Check the 3rd number. If it's bigger than largest, make it the new largest.
  5. Print the largest variable.

Step 4: The Code:

largest_num.cpp 
int num1, num2, num3;
cout << "Enter three numbers: ";
cin >> num1 >> num2 >> num3;

// Assume the first one is the biggest to start
int largest = num1;

// Did num2 beat it?
if(num2 > largest) {
    largest = num2;
}

// Did num3 beat it?
if(num3 > largest) {
    largest = num3;
}

cout << "The largest number is: " << largest;

➕ Example 3: Sum of an Array

Step 1: The Problem: You have an array of 5 numbers. Add them all together and print the total sum.

Steps 2 & 3: The Algorithm:

  1. Create the array of 5 numbers.
  2. Create a sum variable and start it at 0.
  3. Write a loop that spins 5 times.
  4. Inside the loop, grab the current array number and add it to sum.
  5. After the loop is completely finished, print sum.

Step 4: The Code:

array_sum.cpp 
int numbers[5] = {10, 20, 30, 40, 50};
int sum = 0;

// Loop through all 5 items
for(int i = 0; i < 5; i++)
{
    sum = sum + numbers[i];
}

cout << "The total sum is: " << sum;

🗺️ Real-Life Analogy: Planning a Trip

Solving a programming problem is exactly like planning a trip with your friends.

  • Understand: Where are we going? (The goal).
  • Break Down: We need a car, we need snacks, we need a map.
  • Algorithm: "First, we buy snacks. Second, we get in the car. Third, we drive."
  • Code: Actually doing the steps.

If you try to code before you write the algorithm, it's like getting in the car and driving before you even know what city you are going to!

int largest = numbers[0]; // Start by assuming index 0 is biggest

📝 Session Summary

  • Programming is not just about writing code; it's about Problem Solving.
  • Always understand the problem and write an algorithm before you type any C++.
  • An algorithm is just a list of plain-English steps explaining how to solve the problem.
  • Real programs combine multiple concepts. We can put an if statement inside a for loop, and use that loop to read an array!
  • Taking the time to plan your logic saves you hours of fixing broken code later.
