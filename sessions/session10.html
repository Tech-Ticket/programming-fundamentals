🎮 Introduction to the Final Mini Project

Your final mission is to build a Simple Quiz Game.

The program will:

  1. Ask the player a question.
  2. Wait for the player to type an answer.
  3. Check if the answer is correct.
  4. Calculate their total score and print it at the end.

🗺️ Planning the Program (The Algorithm)

Remember Session 9! We never start coding blindly. Let's write out our plan (Algorithm) first.

Our Blueprint:

  1. Start the program.
  2. Create an integer variable called score and set it to 0.
  3. Ask Question 1 to the user.
  4. Get their answer using cin.
  5. If their answer is right, add 1 to score.
  6. Ask Question 2... (repeat the process).
  7. At the very end, print the final score to the screen.

💻 Example Project: Simple Quiz Game

Here is what the basic version of our game looks like translated into C++.

quiz_game.cpp 
#include <iostream>
using namespace std;

int main()
{
    // 1. Setup variables
    int score = 0;
    int answer;

    cout << "--- Welcome to the Math Quiz! ---" << endl;

    // 2. Question 1
    cout << "Question 1: What is 2 + 2?" << endl;
    cin >> answer;

    if(answer == 4)
    {
        score++; // This means score = score + 1
        cout << "Correct!" << endl;
    }
    else
    {
        cout << "Wrong!" << endl;
    }

    // 3. Question 2
    cout << "\nQuestion 2: What is 5 + 3?" << endl;
    cin >> answer;

    if(answer == 8)
    {
        score++;
        cout << "Correct!" << endl;
    }
    else
    {
        cout << "Wrong!" << endl;
    }

    // 4. Final Output
    cout << "\nGame Over! Your final score is: " << score << " / 2" << endl;

    return 0;
}

🔍 Step-by-Step Breakdown

  • int score = 0; starts the player with zero points.
  • We ask the question using cout and listen for their reply using cin >> answer;.
  • The if (answer == 4) checks if what they typed matches the right answer. Notice the double equals sign == for comparing!
  • score++; is a programmer's shortcut. It means "add exactly 1 point to the score variable." It's the same as typing score = score + 1;.

🧱 Real-Life Analogy: Building a House

Programming a big project is like building a house.

  • The Algorithm is the Blueprint. You draw it before you build.
  • Variables are your bricks and wood (your raw materials).
  • If statements and Loops are your tools (hammer and saw) to put the materials together.
  • Functions are the different rooms. Instead of building one massive, messy room, you organize the house into a kitchen, bedroom, etc.

🚀 Improving the Program (Leveling Up)

The code above works perfectly. But real programmers are always looking for ways to make their code cleaner and smarter.

If we had 100 questions, copying and pasting that if/else block 100 times would be terrible! How could we use what we learned in previous sessions to fix this?

  • Use Arrays: We could store all our questions in a string array, and all the correct answers in an int array.
  • Use Loops: We could write a for loop that spins through the arrays, asking the questions automatically.
  • Use Functions: We could put the quiz logic into a function called playQuiz() to keep our main() area perfectly clean.
